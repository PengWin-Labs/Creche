_Session: "Setup scheduled task for chat review automation" · 2026-05-06_

## Decisions

- **Token storage hierarchy: Keychain > env var > `.env` file. Never a plain-text YAML with a custom name.** `config.local.yaml` was the original design — replaced after the user flagged it. The right default for macOS secrets is Keychain (encrypted at rest, locked to login). Env var is the CI fallback. `~/.nightly-retro.env` with `chmod 600` is last resort.
- **One commit per digest file in the git workflow.** Clean PR history: each source gets its own commit on `retro/YYYY-MM-DD`. Reviewer can look at individual files without noise.
- **Idempotent extraction.** If the output file already exists, skip. Running the task twice on the same day produces no duplicates.

## Key Concepts

- **Nightly-retro pipeline shape.** Discover sources (Cowork session transcripts + drop-zone exports) → extract concepts (Haiku API or rules-based fallback) → git branch + commit per file → GitHub PR. The PR is the review surface; not the digest files themselves.
- **Drop zone pattern for mobile chat exports.** `relay/chat-exports/` is a directory that any external process can write `.md` or `.txt` files to. The scheduled task picks them up, moves them to `processed/YYYY-MM-DD/` after handling. Decouples the write side from the processing side.
- **Scheduled task with pre-approval requirement.** First run must be done with the user present so Cowork can prompt for tool-use permissions. Subsequent automated runs inherit those approvals. Pattern: first run is interactive; all later runs are headless.
- **Rules-based extraction as fallback.** When no API key is available (e.g., inside the Linux sandbox), the extractor falls back to heuristic pattern matching. Output is noisier but captures structure. Provides graceful degradation without failing the task.

## Open Questions

- GitHub PR creation requires GITHUB_TOKEN; token is stored in macOS Keychain, inaccessible from the Linux sandbox. Current behavior: files committed locally, no PR created. Workaround needed for the PR step.
